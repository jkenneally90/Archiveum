# Archiveum Library Structure

Archiveum now uses a clearer content-library layout under `archiveum_data/uploads`.

## Factual

- `factual/current_reading`
- `factual/analysis`
- `factual/summaries`
- `factual/reference_notes`
- `factual/research`

Use these for:

- active reading material
- notes you want Archiveum to explain or compare
- summaries and study materials
- factual source documents

## Fiction

- `fiction/prompts/detective`
- `fiction/prompts/mystery`
- `fiction/prompts/humor`
- `fiction/prompts/fantasy`
- `fiction/prompts/science_fiction`
- `fiction/worldbuilding`
- `fiction/characters`

Use these for:

- story starter prompts
- genre-specific writing ideas
- character profiles
- setting notes and worldbuilding references

## Suggested additions

These would fit the project well later:

- `fiction/prompts/horror`
- `fiction/prompts/romance`
- `fiction/prompts/adventure`
- `fiction/plot_outlines`
- `fiction/dialogue_voices`
- `factual/biographies`
- `factual/timelines`
- `factual/project_notes`
- `hybrid/persona_memory`

The `hybrid/persona_memory` idea could be especially useful if you want Archiveum to blend factual recall with creative companion behavior while still keeping the two clearly organized.
